#查看文件的两个版本之间的差异如何帮助你了解引入的错误？

通过比较不同之处，来方便的查看我修改了什么地方，然后再逐一的查看修改的是否正确，找到错误。
---
#从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？

1.如果编写错误了，可以回到之前的版本；2.可以和之前的版本比较，来确认自己修改了那些内容，
以便快速的找到错误的地方。
---
#你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？

手动选择：更灵活；容易忘记，而且麻烦。
自动保存：简单；也许并不是我想要保存的版本。
---
#你如何认为一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？

我认为git保存多个文件更方便查看某一时间点的整体改动。
---
#如何使用命令 git log 和 git diff 来查看文件的历史记录？

文件目录下：git log
文件目录下：git diff ID-1 ID-2
---
#使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？

git checkout 返回原来的版本找到问题后，再用 git checkout 返回现在的版本。
---
#现在你已设置了工作空间，你想尝试使用 Git 来做些什么？

启动atom打开各种代码文件；建立resporters（代码库），利用git log（--stat）管理；
利用git diff查看不同版本的修改内容。
